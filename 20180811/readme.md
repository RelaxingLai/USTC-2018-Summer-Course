	而 `signed char` 用 8 位存储补码

	现给出如下代码段，

	```c
	signed char a = -1;
	unsigned char b = (signed char) a;
	// 将a进行强制类型转换，变成unsigned char，并存到b中
	// 可以理解为将a的数据原封不动地复制给b，只是解读方式变了
	```

	请问，b存储的值是十进制的多少？【255】

	```
	补码 -1 => 1111 1111
	原码 1111 1111 => 255
	```
